In this guide, you'll learn how to build AI agents as Apify [Actors](https://docs.apify.com/platform/actors) in Python, using the agent framework that fits your project.

## Introduction

An [AI agent](https://blog.apify.com/what-are-ai-agents/) gives a large language model (LLM) a goal and a set of tools, then lets the model decide which tools to call, and in what order, until the goal is met. Instead of hard-coding every step, you describe the task in natural language and the model plans the work. An agent framework runs the loop between the model and its tools, so you focus on the prompt and the tools.

The Apify platform is a good place to build and run agents:

- Turn any of the [thousands of Actors](https://apify.com/store) in the Apify Store into an agent tool. Your agent can scrape websites, search the web, or call APIs without you writing the integration.
- Reach a large catalog of models through the [Apify OpenRouter proxy](https://apify.com/apify/openrouter), without managing a provider API key. Token usage is billed to the run's Apify account.
- Store results in a [dataset](https://docs.apify.com/platform/storage/dataset) or [key-value store](https://docs.apify.com/platform/storage/key-value-store), the same as any other Actor.
- Route the agent's browsing through [Apify Proxy](https://docs.apify.com/platform/proxy) and its rotating IP addresses.
- Monetize your agent with the [pay-per-event](../concepts/pay-per-event) pricing model, charging per run or per completed task.
- Schedule runs, expose the agent over the API, and connect it to other apps through [integrations](https://apify.com/integrations).

This guide covers five Python agent frameworks, each backed by an [Actor template](https://apify.com/templates/categories/python) you can start from:

- **[Smolagents](https://huggingface.co/docs/smolagents/index)** - a lightweight framework whose agents write and run Python code to act.
- **[Pydantic AI](https://ai.pydantic.dev/)** - an agent framework that returns validated, typed objects.
- **[LangGraph](https://www.langchain.com/langgraph)** - builds agents as graphs, part of the LangChain ecosystem.
- **[CrewAI](https://www.crewai.com/)** - organizes work as a crew of role-playing agents.
- **[LlamaIndex](https://www.llamaindex.ai/)** - a data framework for LLM apps with built-in agents.

## Before you start

To follow along, you'll need:

- An [Apify account](https://console.apify.com/) and the [Apify CLI](https://docs.apify.com/cli/) installed, so you can run and deploy Actors.
- Python 3.11 or later.
- Access to an LLM, as described next.

### Accessing an LLM

The examples reach their LLM in one of two ways:

- A provider API key. Most frameworks read it from an environment variable, for example `OPENAI_API_KEY`. Keep the key out of the Actor input and source code. On the Apify platform, set it as a [secret environment variable](https://docs.apify.com/platform/actors/development/programming-interface/environment-variables). Locally, export it in your shell.
- The [Apify OpenRouter proxy](https://apify.com/apify/openrouter). This is an OpenAI-compatible endpoint at `https://openrouter.apify.actor/api/v1` that fronts the [OpenRouter](https://openrouter.ai/) model catalog. Token usage is billed to the run's Apify account, so no provider key is needed. The proxy authenticates with the `APIFY_TOKEN` that the platform injects into every run. The Pydantic AI example below uses this approach.

## Using Apify Actors as tools

The main reason to build an agent on Apify is the [Apify Store](https://apify.com/store), with its thousands of ready-made Actors that scrape websites, search the web, and call APIs. Expose one as a tool and your agent can use it without any custom integration. There are two common ways to do this:

- A framework integration. `ApifyActorsTool` wraps a Store Actor as a tool in a single line. It ships in the `langchain-apify` package and works with [LangChain](https://docs.apify.com/platform/integrations/langchain), [LangGraph](https://docs.apify.com/platform/integrations/langgraph), and [CrewAI](https://docs.apify.com/platform/integrations/crewai). LlamaIndex has its own [Apify integration](https://docs.apify.com/platform/integrations/llama-index).
- A plain function tool. Call any Actor with `Actor.apify_client.actor('<actor-id>').call(run_input=...)`, read its dataset, and return the result. The LangGraph and LlamaIndex examples above use this approach. For details, see [Interacting with other Actors](../concepts/interacting-with-other-actors).

## Monetizing with pay-per-event

When you publish your agent, the [pay-per-event](../concepts/pay-per-event) pricing model lets you charge users programmatically from inside the run. This is a good fit for agents, whose cost depends on how much work the LLM and the tools actually do.

You declare the events your Actor can charge for in a `pay_per_event.json` file, then trigger them in code with `Actor.charge`:

```python
# Flat fee when the run starts.
await Actor.charge('actor-start')

# ... run the agent ...

# Flat fee once the task is done.
await Actor.charge('task-completed')
```

The Pydantic AI, LangGraph, CrewAI, and LlamaIndex examples above charge for `actor-start` and `task-completed`. Set the prices to cover your execution and LLM costs, plus a margin. For the full setup, see [Pay-per-event monetization](../concepts/pay-per-event).

## Running on the Apify platform

The agent frameworks are plain Python packages, so the default [Apify Python base image](https://hub.docker.com/r/apify/actor-python) is enough. List `apify` together with the framework packages in your `requirements.txt`, for example for LangGraph:

```text
apify
langgraph
langchain
langchain-openai
```

Provide the LLM credentials in both environments. Set the provider API key as a [secret environment variable](https://docs.apify.com/platform/actors/development/programming-interface/environment-variables) on the platform, and export it in your shell when running locally. If you use the Apify OpenRouter proxy instead, no key is needed, because the run authenticates with the `APIFY_TOKEN` that the platform provides automatically.
